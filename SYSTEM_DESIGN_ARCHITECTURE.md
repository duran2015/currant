# 可鹿 (XiangWang) - 全局系统结构与 Agent 编排方案

## 01 产品定位
**产品名称:** 可鹿 (XiangWang)
**定位:** 基于多智能体协同 (Multi-Agent Orchestrated) 的心理健康支持平台。
**核心价值:** 帮助用户认识自己、理解自己、改善自己。平台不提供医疗诊断，不走社区属性或信息流路线，它是一个“心理驾驶舱”。它的真正护城河是后台隐式运行的 Agent Orchestration 及据此形成的用户心理画像。

## 02 用户画像
**目标用户:** 18-24岁 大学生群体，或者初入职场的新人。
**典型问题:** 考试焦虑、学业压力、失眠、社交压力、就业焦虑、情绪低落、家庭压力。
**设计侧重:** 因为极度敏感，首页呈现极简。不给直接的诊断，而是用 L1、L2、L3、L4 进行隐式兜底。

## 03 Agent 体系 (核心壁垒) ⭐
可鹿放弃传统巨型单体模型，而是采用“多 Agent 编排工作流”。这些 Agent 大部分不直接显示给用户，而是在后台消费各类数据（硬件、标签、对话、量表），并彼此配合。

- **评估Agent**
  - **输入:** 显式量表 (PHQ-9, GAD-7 等)、首次进入的对话分析。
  - **输出:** 用户基础线评估、压力初始等级标签。
  - **记忆:** 长期记忆存储库 (评估时间戳与分值)。
- **咨询Agent**
  - **输入:** 用户的日常语音/文字倾诉。
  - **输出:** 陪伴式聊天、CBT 引导策略、L1 层级情绪舒缓回应。
  - **编排关系:** 在感知到特定深层情绪时，通知“评估Agent”复评，或上报给“风险Agent”。
- **训练Agent**
  - **输入:** 总结Agent 的结论、用户的画像偏好。
  - **输出:** 个性化“今日疗愈任务”（如：5分钟正念、认知写卡）。
  - **流转:** 每天凌晨生成动作并推送到“首页”给用户。
- **追踪Agent**
  - **输入:** 用户的硬件数据（如睡眠、心率等授权数据）、用户的 App 活跃和任务完成情况。
  - **输出:** 情绪波动值、偏离基线的报警。
- **总结Agent**
  - **输入:** 周期内的所有数据（对话日志、评估分、训练完成度、追踪的硬件指标）。
  - **输出:** 并行提炼成阶段性的用户“诊断/画像更新”。为后续的“训练Agent”提供新输入源，达成闭环。在“我的”页面，提供周期复盘文案。
- **风险Agent**
  - **输入:** 咨询Agent或追踪Agent旁路输出的极端敏感词、强波动数据。
  - **输出:** 红色告警干预。
  - **呈现:** 前端拦截其他卡片，置顶强制展示求助热线卡片。

## 04 数据体系 ⭐
**数据源:** 物理硬件数据、量表评估数据、用户显式标签选择、对话文本、应用埋点打卡记录。
**画像中枢 (User Profile):**
- 不断被各类 Agent 更新。
- 维护`基础分`及`短期情绪波动分`。
- 核心标签权重随时间衰减或强化。

## 05 首页 ⭐
首页绝不是信息流，而是 Agent 工作流结果在前端的**仪表盘展示层**。负责：
- 当前状态可视化 -> (来自 *追踪Agent* & *评估Agent*)。
- 今日建议卡 -> (来自 *训练Agent* 输出)。
- 快捷求助/咨询入口 -> (连接 *咨询Agent* 与 人工分诊)。
- 成长连续打卡 -> (由 *总结Agent* 和任务系统记录)。
- 风险提示 -> (由 *风险Agent* 触发，前端强制置顶)。

## 06 AI ⭐
AI 页的核心任务是作为**主要输入源**：
- 表面上是极简的双形态页面 (AI 倾诉 vs 真人倾听师查找)。
- 当用户与 AI 聊天时，后台实则是**咨询Agent**在接待，并将记录推入队列，供**总结Agent**和**追踪Agent**在用户离线后计算，再反哺用户画像。
- 一旦用户需要，直接在该页面发起转介 (通过 L2、L3 引导进入人工预约系统)。

## 07 我的
- 剥离常规设置感，定位于**个人成长中心**:
- 心理画像 (基础与变迁)
- 成长记录 (总结Agent提炼后的用户可见版档案)
- 咨询预约订单记录

## 08 预约
- 将咨询师具象化、真实化，提升转化。
- 加入 AI 预分流建议 (例如“匹配度”)。

## 09 支付
- 提供预约咨询的支付链路、储值或周期陪伴包。

## 10 履约
- **咨询前:** 系统(通过总结Agent)生成脱敏近期画像和周期脉络交于咨询师，极大提升咨询起步效率。
- **咨询中:** 音视频沟通平台，录音(经授权)转文本存放。
- **咨询后:** 咨询师后记汇入系统，并流向“评估Agent”调整下发权重，更新到首页“今日建议卡”。形成大闭环。

## 11 后台
- 患者分流预警台: 接收所有高风险（来自风险Agent）报警。
- 心理干预配置引擎: 设置各种底层知识库给各个 Agent。

## 12 MVP路线图
- **Phase 1: Agent工作流构建** (后台优先，打通 评估->咨询->追踪->风险 的基础调度引擎)。
- **Phase 2: 首页与AI会话交互搭建** (构建端侧基础 UI 和卡片逻辑)。
- **Phase 3: 画像及训练推荐闭环** (结合用户反馈生成周期推荐，完成总结Agent 和训练Agent联调)。
- **Phase 4: C端付费、人工咨询体系与履约** (补充服务能力闭环)。
